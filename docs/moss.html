Get started

Moss is a cross-platform desktop application that is available for Mac OS, Windows and Linux.

Once you have installed it you can create a group, install some tools into the group and start inviting others to join.

If you have received a group invitation, all you need to do is to install the application and click the "join group" button where you paste the link you have been sent.

Start playing together

When you join a group, all of the tools the group is using are waiting for you to use there, you just install the ones you wish to work with.

If you would like to play with another tool with someone in the group you can install that tool from the tool library and it will show up as being in use for others and they can use that one too.

Evolve

Let your groups grow, or shrink over time in both members and complexity.

Add or remove tools as needed, you can also export and import your data if you want to bring some things with you to a new group context.

Moss gives your group the ability to have your collaborations evolve and change along with the group itself!

Tools

By using Moss, you and your group can grow its use of tools over time.

Even just a few people in a group can start collaborating with a tool or play a game togehter, the others don't need to add the complexity if they don't want to use it.

These tools are small applications that are imported into the group. Groups with developer capacities can create and import their own tools too!

There are many tools that have already been created and that are available for anyone who needs them. Available tools include:

Vines: Provides the usual chat functionality: topics, channels, attachments, mentions, etc.
Slate: Provides real-time collaborative drawing. Fully featured diagrams and drawing tools based on the Excalidraw open-source library.
Notebooks: Provides real-time collaborative editing of MarkDown documents. Includes syntax-highlighting, version history, and more.
KanDo: provides robust KanBan functionality with commenting, assignments, labels, categories, checklists and more.
Presence: provides video conferencing with screen sharing
TalkingStickies: provides real-time stickies-boards for ideation, meeting agendas, retros, etc...
Converge: allows stakeholders to shift focus from outcomes to refining a set of criteria that has maximal support. Using this list, they can evaluate and refine proposals to meet those broadly supported criteria.
Who's In?: enables users to propose collective actions or events. With it, anyone can suggest a joint action or event, which other people can agree to on the contingency that every necessary role is filled. This allows group action to grow effortlessly from idea to reality without anyone having to worry about its viability
SnapMail: provides email-like in-group messaging with attachments, replies, etc.
Where: provides a generalized grammar for creating shared maps for groups to see the emergent "whereness" of each other across those them, as well as the grammatics to self-evolved these spaces and how to represent "location" in them.
Gamez: provides a place to design and play arbitrary board games.
WaterTable: provides ability to create database-like tables for collecting and viewing data. Think of this as an "AirTable" clone!
Xylem: provides real-time collaborative spread-sheets based on the Univer open-source library.

Roadmap - Expectations

The Weave and Moss are ambitous efforts, we are making digital spaces for groups to use whatever tools they need. There is a lot of complexity in this and we want to make sure that we find the right people to engage with in the right time.

To help do that, here is our vision of what the different stages or rolling these projects out will look like for the people that are willing to engage.

Germination (WE ARE HERE)

Primarily inviting - Community activators & Developers

Expectations:

  • Many tools already working but with varying degrees of stability
  • Intermittent ability to see each other in groups
  • Data backups need to be taken manually by exporting/importing on a per tool basis
  • There are no always on nodes so for good consistency someone in the group need to keep a computer on and connected
  • Differing levels of maturity of the tools inside
